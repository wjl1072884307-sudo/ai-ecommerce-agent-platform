# 面试演示脚本：AI 电商客服与订单售后 Agent 平台

## 3 分钟演示流程

### 0:00 - 0:30 项目定位

这是一个前后端分离的 AI 电商客服与订单售后 Agent 平台。项目不接真实淘宝、京东或拼多多，而是使用 Mock 商品、订单、物流、售后政策和客服会话数据，模拟真实电商售后场景。

核心目标不是做一个简单聊天机器人，而是展示 Agent 如何接入业务系统，完成意图识别、订单查询、知识库检索、售后规则判断、风险判断、回复建议、人工审核、工单创建和运行日志追踪。

### 0:30 - 1:30 端到端业务演示

1. 打开前端后台。
2. 进入“客服会话”页面。
3. 输入：`我买的耳机有杂音，可以退货吗？`
4. 点击“发送并触发 Agent”。
5. 说明系统完成了：
   - 识别意图为 `return_request`
   - 匹配到耳机订单 `MOCK202606120001`
   - 检索到退货退款政策
   - 判断为疑似质量问题，签收 15 天内可申请退货或换货
   - 由于退货申请需要人工审核，创建审核任务
   - 创建 `return` 类型售后工单
   - 生成客服回复建议

### 1:30 - 2:20 后台页面演示

按顺序打开：

- Dashboard：展示会话数、订单数、待审核、待处理工单、Agent 成功率、意图分布、工单状态。
- 人工审核：查看 Agent 创建的审核任务，并点击通过或驳回。
- 售后工单：查看退货工单，并将状态从 `open` 更新为 `processing`。
- Agent 运行日志：查看 9 个节点的输入输出 JSON，强调可解释和可排障。

### 2:20 - 3:00 技术亮点总结

可以这样总结：

> 这个项目的重点是把 AI Agent 放进真实业务流里，而不是停留在问答层。后端用 FastAPI 和 SQLAlchemy 建模业务数据，用普通 Python Pipeline 实现 Agent MVP，每个节点都有日志。前端用 Vue3 和 Naive UI 做客服后台，可以完整演示从用户消息到审核任务、售后工单和 Agent 可观测日志的闭环。

## 技术亮点

- FastAPI 分层结构：routes、schemas、models、services、agent。
- SQLite + SQLAlchemy：覆盖用户、商品、订单、会话、消息、知识库、Agent run、节点日志、审核任务、工单。
- Agent Pipeline：节点独立、上下文传递、统一日志记录，后续可替换为 LangGraph。
- RAG MVP：知识文档切 chunk，通过关键词匹配退货政策。
- Mock LLM：默认离线可演示，避免 API Key 和网络依赖。
- 可观测性：`agent_node_logs` 保存每个节点的输入、输出、状态、错误和耗时。
- 前端后台：Dashboard、业务管理页、审核页、工单页、日志页完整闭环。
- 自动化测试：覆盖业务逻辑、核心 API 和 Agent 集成链路。

## 常见追问与回答

### 为什么先用普通 Python Pipeline，而不是一开始就用 LangGraph？

MVP 阶段更重视业务闭环和可测试性。普通 Pipeline 更直观，节点函数边界清晰，也方便记录日志。后续如果要支持分支、回退、多 Agent 协作或可视化编排，可以把现有节点迁移到 LangGraph。

### RAG 目前是不是比较简单？

是的，目前是关键词检索版 RAG MVP，目标是先把知识库、chunk、检索和引用链路跑通。后续可以引入 embedding 和向量数据库，支持语义检索、引用高亮和召回评估。

### 为什么使用 Mock LLM？

简历项目和面试演示需要稳定、可复现、无外部依赖。Mock LLM 可以保证离线演示完整流程。生产化时可以抽象 LLM Provider，支持真实模型和 Mock fallback。

### 如何保证 Agent 的结果可解释？

每次 Agent run 都会写入 `agent_runs`，每个节点都会写入 `agent_node_logs`，包括节点名称、输入 JSON、输出 JSON、状态、错误和耗时。前端 Agent 日志页可以查看完整执行链路。

### 如果用户没有明确订单号，如何匹配订单？

MVP 中先按用户订单和商品关键词匹配，例如“耳机有杂音”会匹配耳机订单；没有商品关键词时取最近订单。后续可以加入实体抽取、订单候选确认和多轮追问。

### 项目还能怎么扩展？

- 接入真实 LLM 和 embedding
- 使用 LangGraph 实现可配置工作流
- 引入向量数据库
- 增加登录、JWT 和 RBAC 权限
- 增加工单 SLA 和操作审计
- Docker Compose 一键启动
- 增加客服质检和满意度分析
